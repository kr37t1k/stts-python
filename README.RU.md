# SileroTTS - Стабильный релиз

**README доступен на следующих языках:**

[![RU](https://img.shields.io/badge/RU-red.svg)](https://github.com/kr37t1k/stts-python/blob/main/README.RU.MD)

Silero TTS — это библиотека Python, которая предоставляет простой способ синтеза речи из текста с использованием различных моделей Silero TTS, языков и дикторов. Ее можно использовать как автономный скрипт или интегрировать в ваши собственные проекты Python.

## Особенности

- Поддержка нескольких языков и моделей
- Автоматическая загрузка последнего файла конфигурации модели с логикой повторных попыток
- Предварительная обработка текста и транслитерация
- Пакетная обработка текстовых файлов
- Подробное логирование с помощью loguru
- Отслеживание прогресса с помощью tqdm
- Настраиваемые параметры частоты дискретизации, устройства и многое другое
- Может использоваться как автономный скрипт или интегрироваться в код Python
- Надежная обработка ошибок и проверка
- Устойчивость сети с механизмами повторных попыток

### Особенности API-сервера

- **Веб-интерфейс** - Современный адаптивный интерфейс для генерации TTS
- **REST API** - Полный REST API с несколькими конечными точками
- **Поддержка потоковой передачи** - Потоковая передача аудио в реальном времени
- **История и кэш** - Автоматическое кэширование аудиофайлов и история генерации
- **Аналитическая панель** - Комплексный мониторинг сервера и анализ доступности

- Метрики в реальном времени (время работы, время ответа, процент успешных запросов)

- Интерактивные диаграммы и Графики
- Мониторинг производительности конечных точек
- Анализ ошибок и временная шкала
- Экспорт/импорт данных сессии (JSON/CSV)

- Автоматическое обновление каждые 30 секунд

## Использование

### В качестве Автономный скрипт

Вы можете использовать Silero TTS в качестве автономного скрипта для синтеза речи из текстовых файлов или каталогов, содержащих текстовые файлы.

```
python -m stts [options]
```

#### Параметры

- `--list-models`: Список доступных моделей
- `--list-speakers`: Список доступных дикторов для модели
- `--language LANGUAGE`: Укажите код языка (обязательно)
- `--model MODEL`: Укажите идентификатор модели (по умолчанию: последняя версия языка)
- `--speaker SPEAKER`: Укажите имя диктора (по умолчанию: первый доступный диктор для модели)
- `--sample-rate SAMPLE_RATE`: Укажите частоту дискретизации (по умолчанию: 48000)
- `--device DEVICE`: Укажите используемое устройство (по умолчанию: ЦП)
- `--text TEXT`: Укажите текст для синтеза
- `--input-file INPUT_FILE`: Укажите входной текстовый файл для синтеза
- `--input-dir` `INPUT_DIR`: Укажите входной каталог с текстовыми файлами для синтеза
- `--output-file OUTPUT_FILE`: Укажите выходной аудиофайл (по умолчанию: output.wav)
- `--output-dir OUTPUT_DIR`: Укажите выходной каталог для синтезированных аудиофайлов (по умолчанию: output)
- `--log-level INFO`: Укажите уровень логирования, можно отключить использование значения NONE (по умолчанию: INFO)

#### Примеры

1. Синтез речи из текста:

```
python -m stts --language ru --text "Привет, мир!"

```

2. Синтез речи из текстового файла:

```
python -m stts --language en --input-file input.txt --output-file output.wav
```

3. Синтез речи из нескольких текстовых файлов в каталоге:

```
python -m stts --language es --input-dir texts --output-dir audio
```
### В качестве библиотеки Python

Вы также можете интегрировать Silero TTS в свои собственные проекты Python, импортировав класс `SileroTTS` и используя его методы.

```python
from stts.silero_tts import SileroTTS

# Получение доступных моделей
models = SileroTTS.get_available_models()
print("Доступные модели:", models)

# Получение доступных языков
languages ​​= SileroTTS.get_available_languages()
print("Доступные языки:", languages)

# Получение последней модели для конкретного языка
latest_model = SileroTTS.get_latest_model('ru')
print("Последняя модель для русского языка:", latest_model)

# Получение доступных частот дискретизации для конкретной модели и языка
sample_rates = SileroTTS.get_available_sample_rates_static('ru', latest_model)
print("Доступные частоты дискретизации для последней русской модели:", sample_rates)

# Инициализация объекта TTS
tts = SileroTTS(model_id='v3_en', language='en', speaker='en_2', sample_rate=48000, device='cpu')

# Синтез речи из текста
text = "Hello world!"

tts.tts(text, 'output.wav')

# Синтез речи из текстового файла
# tts.from_file('input.txt', 'output.wav')

# Получение доступных дикторов для текущей модели
speakers = tts.get_available_speakers()
print("Доступные дикторы для текущей модели:", speakers)

# Изменение языка
tts.change_language('en')
print("Язык изменен на:", tts.language)
print("Новый ID модели:", tts.model_id)
print("Новые доступные дикторы:", tts.get_available_speakers())

# Изменение модели
tts.change_model('v3_en')
print("Модель изменена на:", tts.model_id)
print("Новые доступные дикторы:", tts.get_available_speakers())

# Изменение speaker
tts.change_speaker('en_0')
print("Диктор изменился на:", tts.speaker)

# Изменение частоты дискретизации
tts.change_sample_rate(24000)
print("Частота дискретизации изменилась на:", tts.sample_rate)
```

## Возможности CLI

CLI Silero TTS предоставляет следующие возможности:

- **Поддержка языков**: Укажите код языка с помощью флага `--language` для синтеза речи на нужном языке.

- **Выбор модели**: Выберите конкретную модель с помощью флага `--model` или позвольте CLI автоматически выбрать последнюю модель для указанного языка.

- **Выбор диктора**: Выберите диктора с помощью флага `--speaker` или используйте диктора по умолчанию для выбранной модели.

- **Частота дискретизации**: Настройте частоту дискретизации синтезированной речи с помощью флага `--sample-rate`.

- **Устройство**: Укажите устройство (процессор или графический процессор), используемое для синтеза, с помощью флага `--device`.

- **Входной текст**: Укажите текст для синтеза напрямую с помощью флага `--text` или укажите входной текстовый файл с помощью флага `--input-file`.

- **Пакетная обработка**: Обработайте несколько текстовых файлов в каталоге с помощью флага `--input-dir`.

- **Выход**: Укажите выходной аудиофайл с помощью флага `--output-file` или выходной каталог для пакетной обработки с помощью флага `--output-dir`.

- **Список моделей**: Перечислите все доступные модели с помощью флага `--list-models`.

- **Список динамиков**: Перечислите все доступные динамики для конкретной модели с помощью флага `--list-speakers`.

- **Обработка ошибок**: Комплексная обработка ошибок с механизмами повторных попыток для сетевых операций.

## API-сервер

API-сервер SileroTTS предоставляет веб-интерфейс и REST API для генерации речи из текста.

1. Запустите сервер

``batch

python -m stts --server

```

2. Получите доступ к веб-интерфейсу:

- Основной интерфейс: http://localhost:8002

- Информация о моделях: http://localhost:8002/models

- Панель аналитики: http://localhost:8002/data

### Конечные точки API

#### Генерация TTS
- `POST /tts` - Генерация аудио (возвращает JSON в кодировке base64)
- `POST /tts/audio` - Генерация аудио (возвращает аудиофайл)
- `POST /tts/stream` - Ответ потокового аудио
- `GET /tts/audio` - Версия GET для тестирования в браузере
- `GET /tts/stream` - Версия GET для тестирования в браузере

#### Управление
- `GET /models` - Получение доступных моделей и дикторов
- `GET /speakers` - Получить список доступных дикторов для языка/модели
- `GET /settings` - Получить текущие настройки TTS
- `POST /settings` - Обновить настройки TTS
- `GET /history` - Получить историю генерации
- `DELETE /cache` - Очистить аудиокэш
- `GET /cache` - Получить информацию о кэше

### Конечные точки API аналитики

#### Данные аналитики
- `GET /api/analytics` - Получение всех данных аналитики
- `GET /api/analytics/export?format=json|csv&limit=1000` - Экспорт данных (JSON/CSV)
- `POST /api/analytics/import` - Импорт данных сессии
- `DELETE /api/analytics` - Очистка аналитики
- `POST /api/analytics/check` - Ручная проверка доступности записи

#### Состояние и мониторинг
- `GET /api/health` - Базовая проверка состояния (статус, оценка состояния)
- `GET /api/health?detailed=true` - Подробная проверка состояния с метриками
- `GET /api/metrics` - Метрики в стиле Prometheus для мониторинга
- `GET /api/system/info` - Информация о системе (ЦП, память, диск, сеть)

#### Производительность и статистика
- `GET /api/performance` - Подробная статистика производительности
- `GET /api/endpoints` - Статус всех отслеживаемых конечных точек
- `GET /api/history?limit=100&endpoint=/tts/audio` - История запросов с фильтрацией
- `GET /api/stats/hourly?hours=24` - Почасовая агрегированная статистика

### Конечные точки, совместимые с OpenAI

#### Завершение чата (TTS, обернутый в формат чата)
- `POST /v1/chat/completions` - Генерация речи в формате завершения чата
- `GET /v1/voices` - Список доступных голосов/говорящих
- `GET /v1/models` - Список доступных моделей с говорящими (совместимо с OpenAI)

```bash
# Пример завершения чата:
curl http://localhost:8002/v1/chat/completions \

-H "Content-Type: application/json" \

-d '{
"model": "v4_ru_kseniya_v2",
"messages": [
{"role": "user", "content": "Привет, как дела?"}

],

"stream": false
}'
```

#### Список голосов
```bash
# Получить все доступные голоса:
curl http://localhost:8002/v1/voices
```

### Панель аналитики

Панель аналитики обеспечивает комплексный мониторинг производительности и доступности сервера:

**Основные функции:**
- Метрики в реальном времени (время безотказной работы, время ответа, процент успешных запросов)
- Мониторинг системной нагрузки (ЦП, память)
- Тенденции и распределение времени ответа
- Отслеживание производительности конечных точек
- Анализ ошибок и временная шкала
- Статистика объема запросов
- Экспорт/импорт данных сессий
- Автоматическое обновление каждые 30 секунд

**Метрики** Отслеживаемые параметры:**
- Процент времени безотказной работы
- Среднее время ответа
- Показатели успешности/ошибок
- Использование ЦП и памяти
- Частота запросов
- Статистика по каждой конечной точке

**Экспорт данных:**
- Формат JSON для полного сохранения данных
- Формат CSV для анализа в электронных таблицах
- Импорт сессий для сравнения
- Локальное хранилище в браузере

### Пример использования API

```python
import requests

# Генерация речи
response = requests.post(

'http://localhost:8002/tts/audio',
json={

'text': 'Hello, world!',

'language': 'en'

}
)

# Сохранение аудиофайла
with open('output.wav', 'wb') as f:

f.write(response.content)

# Получение аналитики
analytics = requests.get('http://localhost:8002/api/analytics').json()
print(f"Время работы: {analytics['metrics']['uptime']}%")
print(f"Среднее время ответа: {analytics['metrics']['avgResponseTime']}мс")
```

### Пример потоковой передачи

```python
import requests

# Потоковое аудио
response = requests.post(

'http://localhost:8002/tts/stream',
json={

'text': 'Hello, world!',

'language': 'en'

},
stream=True
)

# Воспроизведение аудио в реальном времени
with open('output.wav', 'wb') as f:

for chunk in response.iter_content(chunk_size=8192):

f.write(chunk)
```
